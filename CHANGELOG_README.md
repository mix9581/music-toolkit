# README 重构说明

## 改动目标

让 OpenClaw 等 AI 助手能够清晰理解 music-toolkit 的两种核心使用模式，并正确使用 feishu-toolkit 进行飞书推送。

## 主要改动

### 1. 顶部新增"两种核心模式"章节

在文档开头明确标注：

**模式 1: 歌曲下载模式**
- 用途：下载单曲或歌单的音频文件 + 歌词
- 输出：MP3 文件 + LRC/TXT 歌词文件
- 飞书集成：
  - 小文件 (≤30MB): 直接发送到群聊
  - 大文件 (>30MB): 自动分片上传到飞书云盘 + 发送下载卡片
  - 可选创建歌词文档

**模式 2: 数据抓取模式**
- 用途：抓取歌单或单曲的元数据（播放量、评论数、收藏数等），不下载音频
- 输出：JSON 数据 + CSV 报表
- 飞书集成：发送数据卡片到群聊（表格形式展示关键指标）

## AI 助手使用指南

当 OpenClaw 或其他 AI 助手读取这个 README 后，应该能够：

1. **识别用户意图**：
   - "下载歌曲" → 模式 1
   - "抓取数据" / "查看播放量" → 模式 2

2. **选择正确的命令**：
   - 下载单曲：`download-send`
   - 下载歌单：`download-playlist`
   - 抓取歌单数据：`scrape-playlist`
   - 抓取单曲数据：`scrape-song`

3. **正确使用飞书推送**：
   - 使用 `--send-chat` 参数（需要 feishu-toolkit）
   - 使用 `--webhook` 参数（无需认证）
   - 大文件自动分片上传（无需手动处理）

4. **理解输出格式**：
   - 模式 1：MP3 + LRC + TXT 文件
   - 模式 2：JSON + CSV + 飞书卡片
